## 风险与加速策略

### 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 飞书应用审核延迟 | SSO 无法使用 | Phase 1 就绪后可先切本地账号登录 |
| MES API 对接延迟 | SN 同步不可用 | 手动 CSV 导入 SN 数据作为过渡 |
| 单人开发瓶颈 | 排期延长 | Phase 2 后期加前端 1 人可提前 2 周 |

### 如果想最快上线

**最小可用版本 (MVP): Phase 1 收尾后立即部署** (6/13)

Phase 1 已经包含核心产品管理+项目管理链路，可以作为内部 MVP 先跑起来:
```
1. docker-compose up -d (本地开发)
2. 或者部署到一台 4C8G 云服务器
3. 管理员手动创建用户账号
4. 团队开始录入产品和项目
```

后续 Phase 2-4 按正常节奏开发，每完成一个 Phase 就更新部署。
