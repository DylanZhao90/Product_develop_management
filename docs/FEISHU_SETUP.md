# 飞书开放平台配置实操手册

> 目标：在飞书开放平台创建应用，拿到 SSO 登录和协作功能所需的所有凭证
> 耗时：约 30 分钟

---

## 概览

这个 PDM 系统通过飞书实现了 5 个能力：

| 能力 | 用途 | 所需配置 |
|------|------|---------|
| SSO 扫码登录 | 员工飞书扫码进系统 | OAuth 2.0 网页应用 |
| 审批工作流 | 项目提交飞书审批 | 审批应用 + `PROJECT_REVIEW` 审批定义 |
| 消息机器人 | 任务分配/逾期/认证到期通知 | 机器人能力 |
| 任务中心同步 | WBS 任务同步到飞书任务 | 权限开通 |
| 日历同步 | 里程碑同步到飞书日历 | 权限开通 |

## 第一步：创建飞书企业自建应用

```
1. 打开浏览器进入 https://open.feishu.cn
2. 用安纳瑞企业管理员账号登录（飞书扫码即可）
3. 点击顶部导航「开发者后台」

4. 点击「创建企业自建应用」
   ┌─────────────────────────────────────┐
   │ 应用名称:  纳研 (或 "纳研PDM")       │
   │ 应用描述:  产品研发与生命周期管理平台   │
   │ 应用图标:  上传一个 logo              │
   └─────────────────────────────────────┘

5. 点击「确定创建」
```

```
进入应用详情页 → 左侧菜单「凭证与基础信息」

┌──────────────────────────────────────┐

│ App Secret: xxxxxxxxxxxxxxxxxxxxxx   │  ← 复制，填入 .env 的 FEISHU_APP_SECRET
│                                      │
│ Encrypt Key:                         │  ← 点击「重新生成」,
│   xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx   │     填入 .env 的 FEISHU_ENCRYPT_KEY
```

> **三个值全部复制保存**，App Secret 只显示一次，丢失需重置。

## 第三步：配置网页应用 (SSO 登录)

```
左侧菜单 →「应用能力」→ 点击「添加应用能力」→ 选择「网页应用」

┌─────────────────────────────────────────────┐
│ 网页应用配置                                  │
│                                              │
│ 桌面端主页:  http://localhost:3000            │
│ 移动端主页:  http://localhost:3000            │
│                                              │
│ 保存后，左侧会出现「安全设置」菜单              │
└─────────────────────────────────────────────┘
```

然后配置重定向 URL（OAuth 回调地址）：

```
左侧菜单 →「安全设置」

重定向 URL（回调域名）:
┌──────────────────────────────────────────────────────┐
│ 添加以下 URL：                                        │
│                                                      │
│ http://localhost:3000/auth/callback                   │
│                                                      │
│ （部署到服务器后，追加服务器域名:）                      │
│ https://nayan.anari.com/auth/callback                 │
└──────────────────────────────────────────────────────┘
```

> 我们的前端在 `http://localhost:3000/auth/callback` 接收飞书回调，
> 该页面拿到 code 后调后端 `/api/v1/auth/feishu/callback` 换取 JWT。

## 第四步：添加机器人能力 (消息通知)

```

配置:
┌──────────────────────────────────────┐
```

## 第五步：添加审批能力 (项目审批)

### 5.1 启用审批

```
左侧菜单 →「应用能力」→ 点击「添加应用能力」→ 选择「审批」
```

### 5.2 创建审批定义

进入飞书管理后台（不是开放平台，是企业管理后台）：

```
打开飞书客户端 → 工作台 → 审批 → 管理后台
或浏览器访问 https://www.feishu.cn/approval/admin

→「审批管理」→「创建审批」
┌──────────────────────────────────────────┐
│ 审批名称: 项目立项审批                     │
│                                          │
│ 表单字段（按顺序添加）:                    │
│   1. 项目名称    · 单行文本 · 必填         │
│   2. 关联产品    · 单行文本                │
│   3. 项目类型    · 单行文本                │
│   4. 可行性文档  · 链接/单行文本            │
│                                          │
│ 审批流程: 按需设置（例：直属上级 → 部门负责人）│
└──────────────────────────────────────────┘

保存后，记录审批定义 Code：
  → 审批管理列表 → 找到刚创建的审批 → 查看详情
  → URL 中或详情页会显示审批 Code（默认为自动生成）

  如果 Code 不是 PROJECT_REVIEW，修改代码中的定义：
  backend/app/integrations/feishu/approval.py:16
  "approval_code": "你的审批Code"
```

## 第六步：配置权限

```
左侧菜单 →「权限管理」

搜索并开通以下权限（部分需要管理员审核）:
┌────────────────────────────────────────────┐
│ 权限                      │ 用途           │
│ ──────────────────────────┼─────────────── │
│ 获取用户基本信息            │ SSO 登录       │
│ 获取用户邮箱信息            │ 账号关联       │
│ 获取用户 user_id           │ 用户标识       │
│ 获取通讯录信息              │ 用户搜索       │
│ 获取企业信息                │ Token 签发     │
│                           │               │
│ 审批应用 - 创建/查看审批实例  │ 项目审批       │
│                           │               │
│ 获取与发送单聊/群组消息      │ 机器人通知     │
│                           │               │
│ 任务 - 创建/更新/删除任务   │ 任务同步       │
│                           │               │
│ 日历 - 创建/删除日程       │ 里程碑同步     │
└────────────────────────────────────────────┘

每个权限点击开通后，需要上方的「批量开通」确认。
有「应用身份」和「用户身份」两个开关的，两者都开。
```

## 第七步：配置事件订阅 (审批回调)

```
左侧菜单 →「事件订阅」

1. 开启「事件订阅」开关
2. 请求网址（先填本地能访问的地址，部署后更新）:
   http://<你的开发机IP>:8000/api/v1/callbacks/feishu/event

3. 订阅事件:
   ┌──────────────────────────────────┐
   │ 审批实例状态变更                   │
   │ 审批任务状态变更                   │
   │ 机器人消息事件（如需卡片交互）       │
   └──────────────────────────────────┘

4. 保存 → 飞书会发一个 challenge 请求验证 URL
   （我们的回调代码已实现 challenge 验证，直接保存即可）
```

> 事件订阅的 Encrypt Key 在「凭证与基础信息」页面生成，
> 这就是 `.env` 中的 `FEISHU_ENCRYPT_KEY`。

## 常见问题

**Q: 扫码后没法回到系统？**
检查飞书开放平台「安全设置」→「重定向 URL」是否包含 `http://localhost:3000/auth/callback`

**Q: 提示 App ID 无效 (错误码 60011)？**
检查 `.env` 的 `FEISHU_APP_ID` 是否复制正确，不要有多余空格

**Q: 权限开通后还是报权限不足？**
部分权限有「应用身份」和「用户身份」两个开关，需要两个都开启。开通后必须重新发布应用。

**Q: 审批创建失败？**
确认审批定义 Code 是否与代码中的一致（`PROJECT_REVIEW`），不一致的话修改 `backend/app/integrations/feishu/approval.py` 第 16 行

**Q: 收不到机器人消息？**
机器人发消息给用户需要用户在飞书端先和机器人产生交互（搜机器人名称打开对话窗口），之后才能收到通知。
