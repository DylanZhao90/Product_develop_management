## 2. 核心业务流程 —— 产品从 0 到 1 再到退市

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   [研发中] ─────→ [试产交接] ─────→ [在售] ─────→ [停产] ─────→ [EOL]
      │                │
      └── 打回重研 ────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

阶段          核心业务活动                       PDM 系统支撑功能
────          ────────────                      ──────────────

研发中        ① 产品立项 → 创建产品档案         产品管理 (code 自动生成)
  (核心)      ② 项目审批 → 飞书审批流程         项目管理 (WBS + 飞书审批)
             ③ 需求定义 → WBS 任务分解          项目管理 (任务树 + 交付物)
             ④ 技术方案 → 技术 Issue 跟踪        项目管理 (严重级别 + 状态)
             ⑤ 供应商选型 → 筛选/评估供应商      供应商管理 (资质 + 评价)
             ⑥ 设计开发 → 设计文件版本管理       设计协作 (上传/版本/下载)
             ⑦ 外包管控 → 外包任务 + 交付评审    供应商管理 (外包任务流)

试产交接      ① 试产验证 → 试产阶段项目管理      项目管理 (试产 milestone)
             ② 认证申请 → 认证证书管理           认证管理 (CE/FCC/UL/RoHS)
             ③ 工艺归档 → 设计文件归档           设计协作 (版本固化)
             ④ 试产问题 → 技术 Issue 跟踪         项目管理 (Issue 面板)

在售          ① 产品迭代 → 新版本项目立项         项目管理 (version_upgrade)
             ② 固件发布 → OTA 灰度升级           固件管理 (版本 + 灰度)
             ③ 认证续期 → 到期自动提醒           认证管理 (90/30/7天)
             ④ 产品监控 → 项目健康度分析         分析决策 (Dashboard)

停产          ① 停产决策 → 生命周期流转           产品管理 (→ discontinued)
             ② 资料归档 → 设计文件只读           设计协作 (版本锁定)
             ③ 认证注销 → 认证状态更新           认证管理 (状态流转)

EOL           ① 档案封存 → 全模块只读            全局
  (终态)      ② 审计留存 → 变更历史不可篡改        审计日志
             ③ 知识沉淀 → 产品研发经验库          数据导出

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

关键业务规则:
  · 产品不能跳过试产直接上市 (研发中 → 试产交接 → 在售，不可跳跃)
  · 试产不合格可打回研发 (试产交接 → 研发中，唯一回退路径)
  · 停产不可逆，只能走向 EOL (停产 → EOL，终态不可恢复)
  · 每个项目必须通过飞书审批才能进入执行阶段
```

## 4. 项目管理 —— 系统最复杂的业务域

```
项目全流程: 立项 → 飞书审批 → 执行 → 结项

  pending_approval ──飞书审批通过──→ approved ──手动启动──→ in_progress
       │                                │                      │
       │ 飞书驳回 → closed               │ 取消 → closed         │
       │ 飞书取消 → pending_approval     │                      │
       └────────────────────────────────┘                      │
                                                               ▼
                                                          completed

项目 = 产品研发的一次完整执行单元，包含三个并行维度:

  ┌─────────────────────────────────────────────────────────────┐
  │                     一个项目                                 │
  │                                                             │
  │  📋 WBS 任务树 (结构化执行)    🐛 技术 Issue (异常跟踪)       │
  │  ┌─────────────────────┐    ┌────────────────────────┐     │
  │  │ 里程碑1: 方案设计     │    │ #1 电源效率不达标       │     │
  │  │  ├─ 需求文档 (交付物) │    │    severity: critical  │     │
  │  │  ├─ 原理图设计        │    │    → 指派工程师A        │     │
  │  │  └─ BOM 初稿         │    │    → investigating      │     │
  │  │                      │    │                         │     │
  │  │ 里程碑2: 样机验证     │    │ #2 结构干涉             │     │
  │  │  ├─ PCB Layout       │    │    severity: major      │     │
  │  │  ├─ 结构验证 (外包) ──┼────┼─ 关联外包任务           │     │
  │  │  └─ EMC 预测试       │    │    → resolved            │     │
  │  └─────────────────────┘    └────────────────────────┘     │
  │                                                             │
  │  🏭 供应商外包 (外部协作)                                      │
  │  ┌─────────────────────────────────────────────────────┐   │
  │  │ 外包任务: 结构验证                                    │   │
  │  │   RFQ → 报价(¥12,000) → 交付物提交 → 内部评审 → 通过  │   │
  │  │   供应商: 深圳模具厂 (active · 评分4.5 · 准时率98%)    │   │
  │  └─────────────────────────────────────────────────────┘   │
  └─────────────────────────────────────────────────────────────┘

飞书协作:
  项目审批 → 飞书审批实例 (PM 手机端即可审批)
  WBS 任务 → 飞书任务中心 (截止日期自动同步)
  里程碑   → 飞书日历 (到期提醒)
  任务分配 → 飞书卡片消息 (@被分配人)
  任务逾期 → 飞书催办通知
```

## 6. 固件 OTA 发布流程

```
固件工程师上传 .bin ──→ SHA256 校验 ──→ MinIO 存储 ──→ 版本入库
                                                          │
                          创建 OTA 升级任务 ←──────────────┘
                                 │
                    ┌────────────┼────────────┐
                    ▼            ▼            ▼
                 灰度 10%    灰度 50%    全量 100%
                    │            │            │
                    └────────────┼────────────┘
                                 │
                    ┌────────────┼────────────┐
                    ▼                         ▼
              升级成功 (success_count)   升级失败 (failure_count)
                    │                         │
                    │                   失败原因归因 (JSONB)
                    │                         │
                    └──────────┬──────────────┘
                               ▼
                      飞书通知升级结果

状态: scheduled → in_progress → completed / failed
支持: 灰度比例、按型号/批次筛选目标设备、中途取消
```

## 7. 认证到期管理流程

```
认证创建 ──→ 自动计算到期日 ──→ 定时任务检查 (每天一次)
                                       │
                    ┌──────────────────┼──────────────────┐
                    ▼                  ▼                  ▼
               过期了              即将到期(30天内)      正常
            status = expired    status = expiring_soon     │
                    │                  │                  │
                    │      ┌───────────┘                  │
                    │      ▼                              │
                    │   飞书通知:                          │
                    │   "CE 认证将在 25 天后到期，          │
                    │    请准备续期材料"                   │
                    │                                     │
                    └─────── 飞书通知 ─────────────────────┘
                            "认证已过期"

认证类型: CE / FCC / UL / RoHS
每个认证可独立配置提醒天数 (默认90天)
到期列表 API: GET /certifications/expiring/list?days=90
```

```
  认证→即将到期 ──────────────→│  N  │──→ _on_cert_expiring
                               │  T  │      飞书提醒: "认证 X 天后到期"
  认证→已过期 ─────────────────→│     │──→ 🔴 处理器已定义，未注册
                               │  B  │
  任务→逾期 ──────────────────→│  U  │──→ _on_task_overdue
                               │  S  │      飞书催办: "任务已逾期"

  审批→通过 ──────────────────→│     │──→ _on_approval_approved
                               │     │      飞书通知: "项目已批准"

                               └─────┘

预定义但未接通的事件:
  · firmware.upgrade_completed  → 固件升级完成后通知
  · task.completed              → 任务完成后通知
  · milestone.reached           → 里程碑达成通知
```
